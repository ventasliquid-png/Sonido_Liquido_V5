================================================================================

FLUJO DE INGESTA PDF (Satelite RAR)
------------------------------------
1. Frontend sube PDF via POST /remitos/ingesta-pdf
2. pdf_parser.process_pdf_ingestion() extrae datos fiscales
3. Router verifica cliente por CUIT en BD:
   - Si existe: devuelve datos DB + domicilios con score de matching
   - Si no existe: suggested_action = "CREATE_CLIENT"
4. Usuario confirma datos y envia POST /remitos/ingesta-process
5. RemitosService.create_from_ingestion() ejecuta:
   - Anti-Zombi: verifica duplicado por numero_legal (0016-XXXX-YYYYYYYY)
   - Find Client: busca por ID > CUIT > Razon Social (Fuzzy)
   - Resolve Logistics: domicilio (4 niveles de fallback) + transporte
   - Create Pedido (origen=INGESTA_PDF)
   - Resolve Items: busca producto, si no existe crea VS0001..VS9999
   - Create Remito (numero_legal = 0016-{factura}, aprobado_para_despacho=True)
   - Vanguard Canon: muta flags_estado del cliente (Nivel 13, pierde Virginidad)
   - db.commit()

CONTRATO DE NUMERACION DE REMITOS
----------------------------------
  0015-XXXXXXXX  Serie Manual (Rosa/Blanco sin CAE)
                 Numeracion secuencial desde 00003010
